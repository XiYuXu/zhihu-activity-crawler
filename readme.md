# Zhihu Activity Crawler

一个基于 **Playwright + Python** 的知乎用户历史动态（点赞、收藏、关注问题）的信息抓取工具。  
通过模拟真实用户登录以及访问知乎用户个人主页，脚本自动下拉查阅并刷新内容，解析后将帖子的“标题、链接、点赞数、评论数、点赞或收藏时间”保存为**CSV 和 JSON 文件**

本项目的重点不在于复杂的爬虫技术，绕过反爬虫技术，也不用查阅v3、v4 API，事实上知乎还没有固定的开放API接口，而在于通过 **浏览器自动化 + 页面滚动加载机制识别**，实现稳定的数据获取。

---

# 一、项目重点

知乎用户主页中的动态内容并不是一次性加载完成的，而是采用 **滑动到动态底部后继续刷新出一些新内容** 的方式。

传统的爬虫方式（例如 `requests + API`）在此类网站中通常会遇到以下问题：

- 难以稳定获取真实 API
- 登录 Cookie 和 Token 复杂
- 反爬机制严格
- 接口随时可能变动

因此，本项目采用登录后将cookie保存到本地，然后来直接模拟用户访问网页，让信息内容合法被刷新出来。

---

# 二、方案选择

本项目采用 **Playwright 浏览器自动化框架** 实现数据抓取。

选择这种方式的主要原因包括：

### 1. 完整模拟真实浏览器

Playwright 可以像真实用户一样：

- 打开浏览器
- 访问网页
- 执行 JavaScript
- 滚动页面
- 解析 DOM

这样可以避免处理复杂的接口和加密参数。

---

### 2. 自动管理登录状态

知乎的许多数据需要登录才能访问。

Playwright 支持保存浏览器会话状态：

- 用户只需 **首次手动登录一次**
- 后续运行可以自动复用登录信息

这样可以避免频繁登录带来的问题。

---

### 3. 适应动态加载网页

知乎动态页面使用 **无限滚动加载机制**：

- 页面初始只加载一部分内容
- 当用户滚动页面时，系统才会加载更多数据

Playwright 可以通过 **模拟滚动操作** 来触发新的数据加载。

---


# 三、动态加载识别机制

由于知乎采用 **无限滚动加载（Infinite Scroll）**，程序需要判断：

**什么时候页面已经加载完全部动态。**

本项目采用的方法是：

> 通过监控页面中动态数量的变化来判断是否还有新内容。

基本逻辑：

1. 记录当前页面的动态数量
2. 向下滚动页面
3. 再次统计已获取到的动态数量
4. 判断是否增加

判断规则：

- 如果动态数量 **增加** → 继续滚动
- 如果动态数量 **连续多次未变化** → 认为已加载完所有内容

这种方式可以有效避免无限滚动、和太早停止抓取
---

# 五、数据提取内容

程序会从每条动态中提取以下信息：

| 字段 | 含义 |
|-----|-----|
| 标题 | 动态相关的问题或回答标题 |
| 链接 | 对应内容的访问链接 |
| 点赞数 | 用户点赞数量 |
| 评论数 | 评论数量 |
| 发布时间 | 动态发布时间 |
